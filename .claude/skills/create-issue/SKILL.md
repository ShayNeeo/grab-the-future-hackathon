## Step 4 — Create the issue

Strip the frontmatter from the draft, write body to a temp file, then:

```powershell
$body = Get-Content ".claude\drafts\draft-issue.md" -Raw
$body = $body -replace '(?s)^---.*?---\r?\n', ''
$body | Out-File "$env:TEMP\issue-body.md" -Encoding utf8

gh issue create `
  --repo ShayNeeo/grab-the-future-hackathon `
  --title "<title>" `
  --body-file "$env:TEMP\issue-body.md" `
  --assignee "@me" `
  [--label "<label>"]
```

Capture the returned URL and extract the issue number from it.

## Step 5 — Create the branch

Use `gh issue develop` to create a branch linked to the issue:

```powershell
gh issue develop <N> `
  --repo ShayNeeo/grab-the-future-hackathon `
  --base main `
  --name "<prefix>/#<N>-<slug>" `
  --checkout
```

If `gh issue develop` fails (e.g. no GitHub-linked branch support), fall back to:

```powershell
git switch -c "<prefix>/#<N>-<slug>"
git push -u origin "<prefix>/#<N>-<slug>"
```

**Note:** uncommitted changes on the current branch carry over to the new branch —
warn the user if there are staged/unstaged changes that belong to the previous branch.
