# Open PR — ShayNeeo/grab-the-future-hackathon

**Platform:** GitHub · `gh` · repo `ShayNeeo/grab-the-future-hackathon`\
**Base branch:** `main`\
**PR template:** None (no `.github/pull_request_template.md`) — always use the fallback structure below.\
**Issue-link keyword:** `Closes #<n>` (GitHub closes-on-merge)\
**Title convention:** conventional commits — `type(scope): subject`\
**Reviewers:** Never add automatically. User adds them manually.

Opening a PR is outward-facing — **always confirm before running `gh pr create`.**

## Step 1 — Resolve context

```powershell
git rev-parse --abbrev-ref HEAD   # current branch
git status --short                # uncommitted changes
git push -u origin HEAD           # ensure branch is pushed (run if no upstream)
```

Parse `#<N>` from the branch name (e.g. `chore/#3-wire-result-screen` → 3).
If no issue number in branch name, ask. Use it for `Closes #<N>`.

If the current branch is `main`, stop — nothing to PR from the base branch.

## Step 2 — Build the PR body (grounded in the diff)

Read the diff:

```powershell
git diff main...HEAD --stat
git diff main...HEAD
```

**No PR template exists** — use this fallback structure verbatim, filling every
section with content from the real diff and commits:

```markdown
## Summary

<2–4 sentences: what changed and why. Reference the issue and ScamShield's
purpose (protecting elderly Vietnamese users from scams) where relevant.>

Closes #<N>

## What Changed

- [ ] Flutter UI / screens
- [ ] Flutter state / providers
- [ ] Flutter routing / navigation
- [ ] Backend (scamshield_api)
- [ ] Tests
- [ ] CI / config

*(Tick only what the diff actually touches.)*

## Verification

```
flutter analyze --no-fatal-infos
flutter test
```

<Any manual steps the reviewer should take to confirm the feature works.>

## Breaking Changes

<"None" or list what breaks and how to migrate.>

## Follow-Up

<Optional: list anything intentionally left for a future issue.>
```

Write the filled body to `.claude/drafts/draft-pr.md` with frontmatter:

```
---
title: "<conventional-commit title>"
base: main
head: <branch>
issue: <N>
labels: []
draft: false
---
```

## Step 3 — Resolve title and labels

**Title convention:** `type(scope): subject` (conventional commits)

| Branch prefix | PR type prefix | Example |
|---|---|---|
| `chore/#N-…` | `chore(scope):` | `chore(nav): wire ScamResultCardScreen to route args` |
| `feat/#N-…` | `feat(scope):` | `feat(chat): add voice input via Whisper` |
| `fix/#N-…` | `fix(scope):` | `fix(ui): correct RiskLevel enum mismatch` |
| `docs/#N-…` | `docs(scope):` | `docs(readme): update setup instructions` |

Derive `(scope)` from the changed areas (e.g. `nav`, `chat`, `api`, `ui`).
Offer the conventional form; also accept the issue's `[Tag]` title if the user prefers.

**Labels:** Carry over any labels from the linked issue that exist on the remote.
Available labels: `bug`, `documentation`, `duplicate`, `enhancement`,
`good first issue`, `help wanted`, `invalid`, `question`, `wontfix`.
Skip `task` (doesn't exist). Never auto-create labels.

## Step 4 — Confirm, then create

Show: title · `main ← <branch>` · linked issue (`Closes #<N>`) · labels · ready/draft · body preview.

Get go-ahead. Then:

```powershell
# Strip frontmatter from draft-pr.md
$body = Get-Content ".claude\drafts\draft-pr.md" -Raw
$body = $body -replace '(?s)^---.*?---\r?\n', ''
$body | Out-File "$env:TEMP\pr-body.md" -Encoding utf8

gh pr create `
  --repo ShayNeeo/grab-the-future-hackathon `
  --base main `
  --head <branch> `
  --title "<title>" `
  --body-file "$env:TEMP\pr-body.md" `
  [--label "<label>"] `
  [--draft]
```

Do **not** add `--reviewer` or `--assignee`.
